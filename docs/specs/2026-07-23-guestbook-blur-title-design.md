# NineSense 留言板 BlurText 标题设计

日期：2026-07-23  
状态：设计已确认，等待文档复核

## 1. 背景

留言板当前使用原生 JavaScript 与 CSS 实现的 Shuffle 标题。字符轨道为了维持翻动效果，需要固定字符宽度和裁切窗口，导致最终标题的单词间距不自然，衬线字体也容易出现边角裁切。该效果将被完整移除，标题改为更柔和的 BlurText 入场。

页面仍保持原生 HTML、CSS 和 JavaScript 架构。此次调整只替换标题动画，不引入 React、Motion、打包器或新的运行时依赖。

## 2. 目标

- 保留现有两行标题、字体层级和页面整体布局。
- 按单词播放一次从模糊到清晰的入场动画。
- 恢复自然的英文单词间距，消除字符裁切与机械排列感。
- 在桌面、平板和手机端保持完整显示且不产生横向溢出。
- 在 JavaScript 不可用或用户要求减少动画时直接显示完整标题。

## 3. 非目标

- 不把留言板或主站迁移到 React。
- 不引入 `motion`、GSAP、SplitText 或其他动画依赖。
- 不修改留言表单、公开留言流、后台、API 或数据库。
- 不调整页面配色、背景、标题文案、主区域比例和其他动效。
- 不增加循环播放、鼠标移入重播或手动重播按钮。

## 4. 视觉与动画

主标题固定为两行：

- 第一行：`LEAVE A NOTE`，实心黑色衬线字。
- 第二行：`FOR LATER`，黑色描边衬线字。

动画按以下顺序播放：

`LEAVE` → `A` → `NOTE` → `FOR` → `LATER`

每个单词从上方轻微下落，并经历三个状态：

1. 模糊、透明、向上偏移。
2. 半透明、轻度模糊、轻微越过最终位置。
3. 完全清晰、不透明、回到最终位置。

采用已确认的 A 版节奏：单词间延迟约 115 毫秒，每个单词动画约 720 毫秒。位移和模糊程度以视觉预览为准，初始约为向上 24 像素和 10 像素模糊，中间状态约为向下 4 像素和 5 像素模糊。

动画只在标题首次进入视口时播放一次。完成后标题保持清晰静态，不循环，也不因鼠标移入而重播。

## 5. 实现结构

### HTML

HTML 默认保留完整可读的两行标题和明确的无障碍名称 `LEAVE A NOTE FOR LATER`。每行通过 `data-blur-text` 声明原始文本，确保脚本能够按单词生成动画节点。

### JavaScript

使用单一初始化函数完成以下工作：

1. 读取两行原始文本并按空格拆分为五个单词。
2. 为每个单词创建独立的装饰性动画节点，同时保留标题的无障碍名称。
3. 按两行共享的连续索引设置延迟，保证第二行接在第一行之后播放。
4. 使用 `IntersectionObserver` 在标题首次进入视口时添加播放状态，并立即停止观察。
5. 不注册鼠标重播事件，不建立循环定时器。

当浏览器不支持 `IntersectionObserver` 时立即播放一次。当脚本执行失败时，HTML 中的原始标题仍然可见。

### CSS

CSS 使用关键帧实现透明度、模糊和纵向位移的三阶段过渡。单词节点使用自然内容宽度，通过显式间距保留正常英文词间空白，不再使用固定字符宽度、横向轨道或裁切窗口。

现有 Shuffle 相关选择器、关键帧与稳定状态样式全部删除，避免两套动画规则并存。

## 6. 无障碍与降级

- 标题始终具有完整的可访问名称。
- 动画拆分节点对辅助技术隐藏，避免单词被逐个重复朗读。
- `prefers-reduced-motion: reduce` 下不拆分标题，直接显示完整静态文本。
- JavaScript 未加载时直接使用 HTML 原始标题。
- 动画不改变焦点、不捕获点击，也不影响键盘操作。

## 7. 响应式要求

- 1440px 桌面端保持现有标题尺寸、两列页面比例和两行层级。
- 768px 平板端两行标题完整显示，不与右侧介绍区域重叠。
- 390px 手机端保持自然单词间距，标题不被裁切、不产生横向滚动。
- 动画进行中与结束后都必须满足相同的布局要求。

## 8. 测试与验收

自动化检查必须覆盖：

- 页面包含两行 BlurText 数据源和新的资源版本参数。
- Shuffle 标记、字符轨道、裁切窗口、重播逻辑和相关关键帧已全部删除。
- 正常动画模式生成五个单词节点，顺序与最终标题一致。
- 标题首次进入视口后播放，完成后保持清晰静态。
- 鼠标移出再移入不会重播。
- 减少动画模式下不生成动画节点并直接显示完整标题。
- 1440px、768px 和 390px 视口均无标题重叠、裁切或页面横向溢出。
- 留言提交、后台审核和公开展示完整流程不受影响。

发布后检查公网 HTML、CSS 和 JavaScript 的版本一致性，以及 `/guestbook/` 的 `Cache-Control: no-cache`。保留上一服务器版本作为回滚目标。
