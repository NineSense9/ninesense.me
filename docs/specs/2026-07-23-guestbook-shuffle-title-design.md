# NineSense 留言板 Shuffle 标题设计

日期：2026-07-23  
状态：已确认；2026-07-23 根据线上观感调整标题层级

## 1. 目标

留言板采用英文 Shuffle 标题，同时以克制、稳定的方式保留 NineSense 品牌名。品牌名不再作为大号主标题前奏，避免进入页面时先后出现两组同等级标题造成突兀感。现有资源版本与缓存修复继续保留。

## 2. 已确认的视觉与文案

- 橙色眉题固定为 `NINESENSE / GUESTBOOK / PRIVATE LETTERS`，品牌名始终可见但不抢占主标题层级。
- 页面初次进入时，主标题直接进行横向字符翻动，不再先显示大号 `NINESENSE`。
- 过渡字符仅从 `N`、`I`、`E`、`S` 中取值，不使用数字、符号或乱码字符。
- 最终标题固定为两行：
  - 第一行：`LEAVE A NOTE`
  - 第二行：`FOR LATER`
- 第一行使用实心黑色衬线字，第二行延续现有黑色描边字。
- 最终状态必须保持稳定，不循环播放；鼠标重新移入主标题区域时允许重播一次。
- 桌面端和移动端均不得出现字符重叠、错误换行或横向溢出。

## 3. 实现方式

留言板当前是原生 HTML、CSS 和 JavaScript 页面，不引入 React、GSAP、SplitText 或新的前端构建链。使用原生 DOM、CSS keyframes 和 `IntersectionObserver` 实现与确认稿一致的效果：

1. HTML 默认保留最终英文标题，确保 JavaScript 未加载时仍有完整内容。
2. JavaScript 将最终标题按字符拆分为横向裁切轨道，轨道的过渡字符来自 `NINESENSE` 字母集合。
3. CSS 负责字符轨道从过渡字母平滑归位到最终标题。
4. 动画只在标题首次进入视口时播放；鼠标移出后重新移入可重播。
5. `prefers-reduced-motion: reduce` 下跳过翻动，直接显示最终标题。

主标题使用明确的可访问名称 `LEAVE A NOTE FOR LATER`。眉题中的 `NINESENSE` 作为可见品牌文本保留；动画过程中生成的装饰字符对辅助技术隐藏。

## 4. 缓存修复

用户截图中的异常由新 HTML 与浏览器缓存的旧 CSS/JavaScript 混用造成：旧样式仍使用更大的字号和过紧行距，同时无法识别新标题结构。

本次发布同时采取两层处理：

- 在留言板 CSS 和 JavaScript 地址上增加发布版本参数，确保现有浏览器立即请求新资源。
- 为 `/guestbook/` 增加 `Cache-Control: no-cache`，允许浏览器缓存文件但每次使用前必须向服务器确认版本，避免后续发布再次混用。

后台、API、主页及服务器其他端口的缓存策略不在本次改动范围内。

## 5. 响应式与降级

- 桌面标题上限控制在当前确认稿范围内，行高不低于 `0.96`。
- 768px 以下缩小字号；390px 下两行均必须完整显示且不横向滚动。
- 不支持 `IntersectionObserver` 时直接播放并停在最终状态。
- JavaScript 执行失败时显示静态英文标题，不隐藏核心内容。

## 6. 验收标准

- 首次打开时眉题能清楚读到 `NINESENSE`，主标题直接翻动到最终两行英文标题。
- 页面中不再出现单独占据主标题位置的大号 `NINESENSE`。
- 过渡过程中不出现数字、标点或其他符号。
- 动画结束后标题与确认稿一致，字符不重叠。
- 鼠标重新进入标题区域可以重播，触屏设备只播放首次入场。
- 减少动画模式下直接展示最终标题。
- 1440、768、390 三种视口无横向溢出，标题两行边界不相交。
- 公网响应使用新资源版本参数，并返回预期的缓存响应头。
- 留言提交、后台审核及公开展示流程不受影响。

## 7. 验证范围

- 静态发布契约：标题文案、字符集、版本参数、减少动画规则。
- 部署契约：`/guestbook/` 缓存响应头。
- 浏览器回归：固定品牌眉题、Shuffle 最终状态、重播、响应式、不重叠与完整留言流程。
- 公网发布后检查：HTML/CSS/JavaScript 版本一致、服务健康、桌面与手机截图。
