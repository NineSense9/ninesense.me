# NineSense 留言板首版设计

日期：2026-07-23  
状态：已于 2026-07-23 通过文档复核  
目标站点：NineSense 个人网站

## 1. 目标

为 NineSense 增加一个可以长期维护的独立留言板。访客无需注册，可以发布待审核的公开留言，也可以发送永不公开的私信。站长使用单一管理账户审核、回复和归档内容，并在收到新内容后获得邮件提醒。

首版优先保证：隐私边界明确、服务器资源占用低、移动端可用、数据可备份、后续可以扩展到项目和网站内容管理。

## 2. 已确认的产品规则

- 使用一个独立的 `/guestbook/` 页面。
- 发布区同时支持“公开留言”和“私信给我”两种类型。
- 访客无需注册或登录。
- 昵称必填；联系方式选填，可填写邮箱或手机号码。
- 联系方式仅在管理后台可见，任何公开页面和公开接口均不得返回。
- 所有公开留言先审核，审核通过后才展示。
- 私信永不公开，也不提供误操作为公开内容的入口。
- 站长可以在后台公开回复已通过的留言，回复显示在原留言下方。
- 新留言或私信进入系统后向站长发送邮件提醒。
- 首版只支持纯文本和普通 emoji，不支持图片、附件、Markdown 或 HTML。
- 正常访客不显示验证码；使用频率限制、隐藏陷阱字段、内容限制和服务端检查防止垃圾信息。
- 首版只有一个管理账户，不开放注册；该账户以后可以扩展为网站内容管理账户。

## 3. 不在首版范围内

- 访客账户、社交登录和访客自助编辑或删除。
- 多管理员、角色权限和审批流。
- 访客之间的多层回复或讨论串。
- 图片、文件、富文本、Markdown、自定义 HTML。
- 点赞、表情反应、关注、站内通知和实时聊天。
- 自动公开、AI 自动回复或 AI 自动审核。
- 面向大规模流量的分布式数据库和多机部署。

## 4. 页面与交互设计

### 4.1 页面结构

- 主页导航增加“留言板”入口，指向 `/guestbook/`。
- 页面沿用现有暖白、黑色和琥珀色体系，使用较短的页首，不复制主页的大型 Hero。
- 桌面端采用“左写右看”：左侧为发布表单，右侧为公开留言流。
- 平板和手机端改为上下布局，先显示发布表单，再显示留言流。
- 发布区使用明显的“公开留言 / 私信给我”切换；私信模式必须明确说明内容不会公开。

### 4.2 表单

- 昵称：必填，去除首尾空白，长度 2–24 个可见字符。
- 联系方式：选填，接受一个有效邮箱地址或手机号码，最长 100 个字符。
- 内容：必填，去除首尾空白，长度 2–500 个可见字符。
- 隐藏陷阱字段：正常用户不可见；机器人填写后请求被静默拒绝。
- 表单显示剩余字数，允许普通 Unicode emoji。
- 所有用户内容仅按纯文本处理和渲染。

### 4.3 提交反馈

- 成功后统一提示：“已经收到，公开留言会在审核后出现。”
- 私信成功提示补充“这条内容只会在后台显示”。
- 页面不向访客暴露具体审核状态、拦截规则或邮件发送结果。
- 网络失败时保留全部已输入内容并允许重试。
- 字段错误在对应字段附近显示；频率限制使用自然语言提示稍后再试。
- 前端为一次提交生成幂等标识，避免网络重试产生重复记录。

### 4.4 公开留言流

- 只展示状态为 `published` 的公开留言。
- 按审核通过时间倒序排列，而不是按提交时间排列。
- 每次读取 10 条，使用稳定游标分页；“继续看看”加载下一页。
- 对外字段仅包含：留言 ID、昵称、展示日期、正文、站长公开回复及回复日期。
- 日期显示到日，不展示精确时间、联系方式、内部状态或访客网络信息。
- 撤回公开的留言必须立即从公开接口消失。

### 4.5 可访问性

- 所有控件支持键盘操作和清晰焦点样式。
- 标签和错误信息与字段建立可访问关系。
- 支持 `prefers-reduced-motion`，动效不会阻碍填写与阅读。
- 桌面、平板和手机均不得出现横向溢出。

## 5. 系统架构

### 5.1 组件

1. 现有静态站点继续由 Nginx 提供。
2. 新增独立的 Python API 服务，使用 FastAPI、SQLAlchemy 2 和 Alembic。
3. API 仅监听 `127.0.0.1:8812`，不直接暴露到公网。
4. Nginx 将 `/api/` 转发到 API，并提供 `/guestbook/` 与 `/admin/` 静态页面。
5. API 由 systemd 托管，异常退出后自动恢复。
6. SQLite 数据库存放在发布目录之外，网站版本切换不得覆盖数据。

### 5.2 服务器路径

- 应用代码：`/opt/ninesense-guestbook/releases/<timestamp>`。
- 当前版本链接：`/opt/ninesense-guestbook/current`。
- 数据库与共享数据：`/var/lib/ninesense/guestbook.sqlite3`。
- 私密配置：`/etc/ninesense/guestbook.env`，权限仅限服务账户和 root。
- 备份：`/var/backups/ninesense/guestbook/`。

### 5.3 项目整理

当前发布源分散在一次性工作目录中且没有 Git 仓库。实现阶段应将已验证的静态发布包复制到当前 `ninesense.me` 项目根目录，建立清晰的 `site/`、`server/`、`tests/`、`deploy/` 和 `docs/` 边界。复制和验证完成前不删除原发布源，也不覆盖服务器当前版本。

## 6. 数据设计

### 6.1 留言

每条留言至少包含：

- 随机不可枚举 ID。
- 类型：`public` 或 `private`。
- 昵称。
- 加密联系方式、联系方式类型与加密版本；未填写时为空。
- 纯文本正文。
- 状态：`pending`、`published`、`handled`、`archived`、`rejected`。
- 提交、审核、公开、处理、归档和更新时间。
- 站长公开回复与回复时间；私信不得拥有公开回复。
- 幂等标识，用于去除重复提交。

公开留言的合法状态流转：

`pending → published → archived`  
`pending → rejected`  
`published → pending` 用于撤回公开后重新处理。

私信的合法状态流转：

`pending → handled → archived`  
`pending → rejected`

彻底删除是显式管理动作，不作为普通状态。

### 6.2 管理账户与会话

- 仅初始化一个管理账户，不提供公开注册接口。
- 密码使用 Argon2id 哈希。
- 密码重置通过服务器命令完成，不使用公开找回页面。
- 会话 ID 使用高强度随机值；数据库只保存会话 ID 的哈希。
- Cookie 使用 `HttpOnly`、`SameSite=Strict`，正式 HTTPS 上线后强制 `Secure`。
- 会话默认 8 小时过期，支持主动退出和服务端撤销。

### 6.3 联系方式加密

- 联系方式使用 AES-256-GCM 应用层认证加密。
- 加密密钥只存在于私密环境配置，不进入数据库、代码仓库、日志或备份包。
- 密文保存加密版本与随机 nonce，支持以后轮换密钥。
- 只有通过管理会话验证的后台接口可以请求解密。

### 6.4 邮件发件箱

- 留言入库与提醒任务创建在同一个数据库事务中完成。
- 后台任务从发件箱读取并发送 SMTP 邮件。
- 失败后指数退避重试，并在管理后台显示提醒状态。
- 邮件只包含类型、昵称、提交时间、正文摘要和后台入口；不包含完整联系方式。

## 7. API 边界

公共接口：

- `GET /api/guestbook/messages`：游标分页读取已公开留言。
- `POST /api/guestbook/messages`：提交公开留言或私信，始终先进入待处理状态。
- `GET /api/health`：只返回服务是否正常，不暴露环境信息。

管理接口：

- 创建、读取和删除管理会话。
- 按状态和类型分页读取留言。
- 查看单条留言与解密后的联系方式。
- 审核公开、撤回、拒绝、标记处理、归档和彻底删除。
- 新增、修改和删除公开回复。
- 查看邮件提醒状态并手动重试。

所有管理写操作必须经过管理会话验证与 CSRF 校验。跨域请求默认不开放，页面与 API 使用同源路径。

## 8. 审核后台

- 默认进入“待审核”列表。
- 分类包括：待审核、已公开、私信、已归档、已拒绝。
- 支持按昵称和正文搜索；联系方式不参与普通搜索。
- 公开留言支持：通过、通过并回复、修改回复、撤回公开、归档、拒绝、彻底删除。
- 私信支持：查看联系方式、标记已处理、归档、拒绝、彻底删除。
- 私信界面不渲染“公开”操作。
- 高风险操作需要二次确认；批量操作不进入首版。

## 9. 防垃圾与安全

- Nginx 和应用层共同限制提交频率与请求体大小。
- 应用层使用服务端密钥对访客 IP 生成短期不可逆标识；不长期保存原始 IP。
- 频率记录和失败登录记录自动到期清理。
- 隐藏陷阱字段、最短填写时间和幂等标识用于拦截常见机器人与重复提交。
- 所有字段在服务端再次校验；所有查询使用参数绑定或 ORM。
- 前端使用 `textContent` 等纯文本方式渲染访客内容，服务端禁止 HTML。
- 管理登录限制连续失败，并记录不含敏感数据的安全事件。
- 管理写操作使用 CSRF 防护；Cookie、CSP、`X-Content-Type-Options`、`Referrer-Policy` 等安全头保持启用。
- 日志不记录完整正文、联系方式、密码、会话值、密钥或原始 IP。

## 10. 异常处理与保留策略

- 数据库启用 WAL、外键、事务、合理的忙等待和有限重试。
- 数据库持续忙碌时返回可恢复错误，不进行不确定的重复写入。
- SMTP 故障不影响留言保存，提醒任务保留并重试。
- 被拒绝内容保留 30 天后自动清理。
- 已公开留言、公开回复和归档私信不自动删除，由站长决定。
- 过期会话、频率记录、失败登录记录和已成功邮件任务定期清理。

## 11. 备份与恢复

- 使用 SQLite 在线备份机制或一致性备份命令，不能在 WAL 写入期间直接复制主文件充当备份。
- 每天自动备份，并保留滚动历史；首版目标为 30 份日备份和 12 份月备份。
- 备份文件权限仅限服务账户与 root。
- 定期在临时位置执行完整性检查，确认备份可以打开并包含最新迁移版本。
- 数据库迁移前必须创建即时备份；迁移脚本需要明确升级与回退策略。

## 12. 测试策略

### 12.1 单元测试

- 昵称、联系方式和内容校验。
- 联系方式加密、解密、错误密钥和密钥版本。
- 留言状态机与非法状态转换。
- 频率限制、幂等提交、邮件重试和清理策略。

### 12.2 API 集成测试

- 公开留言与私信提交。
- 未审核、私信和联系方式无法从公开接口读取。
- 游标分页稳定且无重复或遗漏。
- 登录、退出、会话过期、CSRF 和失败登录限制。
- 审核、公开回复、撤回、归档、拒绝和删除。
- SMTP 与数据库暂时故障时的行为。

### 12.3 安全测试

- XSS、SQL 注入、超长载荷、非法 Unicode、重复提交和伪造管理请求。
- Cookie 和响应安全头。
- 公共接口字段白名单，防止模型或序列化误带敏感字段。
- 日志扫描，确保没有联系方式、凭据或会话值。

### 12.4 浏览器测试

- 1440、768 和 390 像素宽度下无横向溢出。
- 公开与私信切换、提交反馈、错误后内容保留和继续加载。
- 后台登录、审核与回复完整流程。
- 键盘操作、焦点样式和减少动效。

### 12.5 部署验证

- API 健康检查、systemd 状态、Nginx 反向代理和请求体限制。
- 当前公网 IP 与 8811 端口上的完整流程。
- 首页、留言板、后台、关键接口与 404 响应。
- 创建测试留言、审核公开、撤回并删除的端到端烟雾测试。
- 从备份副本恢复并执行数据库完整性检查。

## 13. 发布与回滚

1. 在本地完成自动化测试和三种视口检查。
2. 将后端部署为独立版本，不修改当前静态站点入口。
3. 创建数据库与管理账户，验证备份任务和邮件配置。
4. 在当前 IP 与 8811 端口测试 `/guestbook/`、`/admin/` 和 `/api/`。
5. 稳定后再向主页导航加入留言板入口。
6. 域名与 HTTPS 接入只修改 Nginx 入口，不改变页面和 API 路径。

代码回滚通过版本目录与 `current` 链接完成。数据库迁移前创建即时备份；若回滚代码无法兼容新结构，先执行已验证的数据库回退迁移或恢复即时备份。任何回滚不得删除已经接收的留言而不事先备份。

## 14. 验收标准

- 无账户访客可以提交公开留言或私信。
- 联系方式在公开页面、公开 API、邮件提醒和日志中均不可见。
- 未审核公开留言和所有私信均无法从公开接口读取。
- 管理员可以完成审核、回复、处理、归档、拒绝和删除。
- 新内容可以触发可重试邮件提醒，邮件故障不丢留言。
- 常见机器人提交、重复提交和暴力登录受到限制，正常访客无需验证码。
- 页面在桌面、平板和手机上可正常使用，无横向溢出。
- 服务重启、代码发布和数据库备份不会丢失已存数据。
- 能从备份副本恢复数据库并通过完整性检查。

## 15. 已知前置条件

- 正式域名接入仍等待 ICP 备案流程；功能开发和 IP 端口测试不依赖备案完成。
- 邮件提醒需要站长提供一个可用 SMTP 账户及应用专用密码，凭据只写入服务器私密配置。
- 正式 HTTPS 上线前，管理后台仅用于受控测试；生产管理登录必须使用 HTTPS。
