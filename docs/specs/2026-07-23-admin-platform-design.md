# NineSense 管理平台设计

- 状态：设计已确认，文档待用户复核
- 日期：2026-07-23
- 适用范围：NineSense 公开网站、管理平台、内容发布、互动、统计与运维

## 1. 背景与目标

NineSense 是一个长期维护的实名个人网站。现有公开页面为轻量静态站点，留言板使用 FastAPI、SQLAlchemy 和 SQLite，已有登录、留言审核、私信处理、公开回复与邮件重试能力。

本次扩展的目标是在不破坏现有网站视觉和访问稳定性的前提下，将现有留言后台发展为完整的个人网站管理平台。平台需要覆盖内容、媒体、页面、互动、发布、统计、安全与运维，同时保持单人维护成本可控。

核心原则：

1. 公开网站不依赖后台实时运行。
2. 所有有意义的公开内容都能在后台维护。
3. 内容使用单语言字段，不恢复中英文切换、双语字段或自动翻译。
4. 先预览、后发布；发布失败不得破坏当前线上版本。
5. 数据使用开放格式，可备份、验证、恢复和迁移。
6. 默认保护访客隐私，不建立个人画像。
7. 公开仓库不得包含生产凭据、访客隐私或服务器敏感信息。

## 2. 非目标

- 不提供任意 CSS、任意 JavaScript 或完全自由画布式页面设计。
- 不为访客建立账户系统。
- 不在当前阶段建设多管理员和复杂角色权限。
- 不在后台提供从 GitHub 自动更新生产服务器的按钮。
- 不将视频文件直接上传到当前服务器。
- 不将公开网站改造成依赖后台接口实时渲染的动态站点。

## 3. 总体架构

系统采用“管理后台保存结构化数据，发布时生成静态快照”的混合架构。

### 3.1 公开网站

- 继续使用静态 HTML、CSS 和 JavaScript，保留现有布局、动效和响应式行为。
- 页面读取同一发布版本下的静态内容快照和媒体清单。
- 已发布快照由 Nginx 直接提供，FastAPI 停止时公开网站仍可访问。
- 快照按版本保存，通过原子切换更新当前版本，禁止覆盖式发布。

### 3.2 管理前端

- 管理后台升级为组件化单页应用，面向复杂表单、可视化编辑、拖拽排序和实时预览。
- 桌面端为主要工作环境，平板端可完成常规操作。
- 手机端保证登录、通知查看、互动审核和紧急下线可用，不将复杂页面搭建作为手机端验收目标。
- 管理前端只通过版本化管理 API 读写草稿和管理数据，不直接修改生产静态文件。

### 3.3 后端

FastAPI 后端按职责拆分为以下模块：

- 身份与会话
- 内容库
- 页面配置
- 媒体处理
- 发布与版本
- 留言、私信与评论
- 通知与后台任务
- 访问统计
- 备份与恢复
- 运维状态与审计

模块通过明确的服务接口协作，不在路由或单个文件中混合数据库、文件处理、发布和通知逻辑。

### 3.4 数据与任务

- 继续使用 SQLite，并启用 WAL、外键约束、忙等待和定期完整性检查。
- 数据库只存储结构化数据、草稿、版本元数据、审计记录与任务状态。
- 原始媒体、派生媒体、发布快照和备份放入职责分离的文件目录。
- 邮件、媒体转换、统计汇总、清理和备份使用数据库任务队列与系统定时服务，不引入 Redis 或独立消息中间件。

## 4. 后台信息架构

后台使用固定主导航，包含以下入口：

1. **总览**：待处理互动、待发布修改、近期访问、服务状态、备份状态和异常提醒。
2. **内容库**：个人资料、项目、算法与比赛、Agent 开发、科研、奖项、时间线、文章和外部链接。
3. **页面管理**：页面列表、结构化模块、排序、预览和页面级设置。
4. **媒体库**：图片、PDF、外部视频、引用、版本和清理。
5. **互动中心**：留言、私信、评论、回复、审核与垃圾内容。
6. **发布中心**：待发布更改、发布检查、预览、发布记录、差异和回滚。
7. **访问统计**：趋势、热门内容、来源、设备与导出。
8. **运维中心**：服务、任务、邮件、存储、备份、恢复与异常。
9. **设置与安全**：站点、导航、页脚、SEO、域名、邮件、验证码、账户、两步验证和日志。

## 5. 内容模型

### 5.1 通用字段

所有内容共享以下基础信息：

- 唯一标识和内容类型
- 标题、摘要与正文
- 稳定地址标识（slug）
- 时间、排序、标签和可见性
- 封面、媒体引用和外部链接
- 草稿状态、公开状态和归档状态
- SEO 标题、说明、分享图和索引策略
- 创建、修改、发布和归档时间

内容地址一旦公开，修改 slug 时必须保留旧地址重定向，避免搜索结果和外部链接失效。

### 5.2 内容类型

- **个人资料**：简介、当前关注方向、联系方式入口与社交平台。
- **项目**：背景、目标、个人职责、技术栈、过程、结果、截图、演示和相关链接。
- **算法与比赛**：赛事、阶段、时间、个人贡献、结果、奖项和复盘。
- **Agent 开发**：工作方式、工具组合、任务拆分、测试、部署和经验总结。
- **奖项**：名称、级别、时间、说明、证书图片和关联经历。
- **时间线**：年月、标题、说明、关联内容和展示优先级。
- **文章**：摘要、正文、封面、发布时间、外部原文和评论开关。
- **外部链接**：GitHub、Gitee、CSDN 等平台的名称、地址、说明和排序。

### 5.3 科研模块

科研作为独立顶层内容类型，预留：

- 研究方向与兴趣
- 科研项目
- 论文与预印本
- 数据集与开源代码
- 学术会议、汇报与海报
- 合作成员与个人贡献
- DOI、论文、数据和代码链接
- 准备中、投稿、在审、录用和发表状态

科研内容为空时，公开导航、页面模块和详情入口自动隐藏，不展示空栏目。

### 5.4 编辑器

- 默认提供可视化富文本编辑，同时支持 Markdown 源码模式。
- 支持 Markdown 导入、导出、粘贴和图片粘贴。
- 编辑结果保存为可验证的结构化文档，同时可稳定转换为 Markdown 和安全 HTML。
- 自动保存服务器草稿，并在浏览器保留短期本地恢复稿。
- 使用内容修订号进行并发检查，旧页面保存时不得静默覆盖新版本。

## 6. 页面搭建器

页面由结构化模块组成。首批模块包括：

- 首屏
- 个人介绍
- 精选项目
- 算法与比赛
- Agent 开发经历
- 科研
- 奖项展示
- 时间线
- 文章列表
- 图片组
- 文字与媒体
- 外部链接
- 留言入口

每个模块支持新增、复制、隐藏、删除、拖拽排序、锚点、内容来源和布局方案。内容来源可以选择具体记录，也可以按类型、标签和时间自动选取。

设计控制限制在经过验证的布局变体、文字与图片比例、间距、背景和对齐方式内。模块不得注入任意 CSS 或脚本。后台提供桌面、平板和手机三种预览，现有主页迁移为第一份页面配置。

## 7. 媒体库

### 7.1 图片

- 上传后检查扩展名、真实文件类型、像素、文件大小和解码结果。
- 重新编码并移除不需要的元数据。
- 自动生成适合不同屏幕的多尺寸 WebP 成品，并保留受保护的原图。
- 原图不经过公开路由提供；公开页面只使用派生文件。
- 每张图必须能够填写 ALT 文本、说明、来源和关联内容。

### 7.2 版本与清理

- 替换图片时保留最近 3 个旧版本。
- 旧版本超过 30 天后进入清理任务，但当前引用版本不得清理。
- 删除前显示所有引用位置；存在有效引用时阻止删除。
- 未使用文件先进入回收站，经过保留期后再彻底清理。

### 7.3 PDF 与视频

- PDF 可以上传、下载和关联内容，但必须经过类型、大小和安全响应头检查。
- 视频只保存可信外部链接或嵌入地址，不上传到当前服务器。

## 8. 草稿、预览与发布

### 8.1 草稿

- 编辑内容和页面时自动保存草稿，不直接影响线上内容。
- 发布中心集中显示全部未发布修改，并按新增、修改、隐藏和删除分类。
- 每次保存保留修订记录，历史记录包含操作者、时间和修改摘要。

### 8.2 发布检查

发布前必须检查：

- 必填字段
- 重复或非法 slug
- 失效内部引用
- 缺失、待删除或转换失败的媒体
- 缺少 ALT 的公开图片
- 无效外部链接
- 页面模块配置与响应式约束
- 静态快照能否完整生成

阻断性错误必须修复后才能发布；非阻断性提醒允许站长明确确认后继续。

### 8.3 预览与发布

- 预览读取草稿快照，并通过短期、仅登录站长可访问的地址展示。
- 发布前填写简短发布说明。
- 后端在隔离目录生成完整静态快照，校验成功后记录版本并原子切换当前版本。
- 任一步骤失败都清理未完成产物，并保持原版本在线。

### 8.4 历史与回滚

- 发布记录包含版本号、说明、内容清单、媒体清单、校验结果和发布时间。
- 后台可以比较相邻版本的结构化差异。
- 回滚先验证目标快照完整性，再将其作为一次新的发布记录上线；不删除中间历史。

## 9. 互动中心

留言、私信和内容评论采用统一数据模型，并保留各自的展示与状态规则。

- 访客无需注册。
- 昵称和正文必填，邮箱或手机选填且永不公开。
- 评论绑定具体内容；每条内容可独立开关评论，并提供全站紧急关闭开关。
- 新互动默认待审核，支持公开、回复、撤回、已处理、归档、拒绝、垃圾标记和回收站。
- 后台支持搜索、筛选、批量操作和仅站长可见的内部备注。
- 回收站保留 30 天；永久删除同步销毁联系方式密文和关联任务。

防滥用采用渐进策略：隐藏字段、最短填写时间、频率限制和行为判断默认无感运行，只对可疑请求显示验证码。系统不保存原始 IP，只使用带服务端密钥、短期轮换的不可逆标识，并按保留策略删除。

公开内容与站长回复均进行严格 HTML 清理和输出转义，禁止脚本、事件属性和危险链接协议。

## 10. 通知

- 后台通知中心即时记录普通互动、发布、任务和系统事件。
- 普通留言与评论可按配置合并为邮件摘要。
- 备份失败、发布失败、存储不足、服务异常和连续登录失败立即发送邮件。
- 邮件使用现有 Outbox 思路扩展为通用通知队列，提供退避重试、失败原因和手动重试。
- 接收地址、摘要频率、通知类别和免打扰时段可配置。

通知日志不得保存 SMTP 密码、完整联系方式、会话或邮件正文中的敏感字段。

## 11. 访问统计

- 采用自建、无 Cookie、无广告追踪、无访客指纹的轻量统计。
- 记录页面、时间、来源站点、设备类别和机器人判断所需的最少字段。
- 不提供单个访客轨迹，不保存原始 IP。
- 详细事件保留 90 天，之后只保留按日、页面、来源和设备汇总的数据。
- 支持排除已知机器人和站长访问，支持导出 CSV。

## 12. 身份、安全与审计

### 12.1 账户

- 当前只启用一个站长账户，数据结构预留未来增加账户的可能。
- 密码使用成熟的自适应密码哈希。
- 登录加入基于 TOTP 的两步验证和一次性恢复码。
- 恢复码只显示一次，以不可逆形式保存，使用后立即失效。
- 支持查看并撤销所有活动会话。

### 12.2 高风险操作

修改密码、关闭两步验证、生成新恢复码、查看联系方式、永久删除、恢复备份和修改关键安全设置需要重新验证身份。正式后台只允许 HTTPS，使用安全 Cookie、CSRF、防点击劫持、内容安全策略和登录限速。

### 12.3 审计

以下事件写入只追加的审计记录：

- 登录成功、失败、退出和会话撤销
- 两步验证和账户安全修改
- 查看私密联系方式
- 内容、页面、媒体与设置修改
- 发布、回滚、删除、备份和恢复
- 通知重试与运维操作

审计日志只保存必要的事件类型、结果、时间和内部对象标识，不保存密码、Cookie、会话值、原始 IP、联系方式正文、加密密钥或 SMTP 密码。

## 13. 运维、备份与恢复

### 13.1 运维状态

后台展示公开站点、API、数据库、邮件队列、定时任务、存储空间、HTTPS 证书、备份和当前发布版本。每项状态包含最后成功时间、失败原因和可执行建议。

### 13.2 备份

备份覆盖：

- 数据库一致性快照
- 原始媒体和必要派生清单
- 已发布静态快照与版本清单
- 非密钥类站点配置
- 恢复所需的结构版本信息

本地每日备份保留 30 天，每月备份保留 12 份；同时生成加密异地备份。异地存储凭据、加密密钥和恢复密钥不写入公开仓库，也不与备份明文放在同一位置。

平台提供 JSON、Markdown、原始媒体和引用清单组成的开放格式导出包，避免未来被当前实现锁定。

### 13.3 恢复

恢复流程必须：

1. 重新验证站长身份。
2. 创建恢复前即时备份。
3. 在隔离目录解密并校验备份。
4. 检查数据库完整性、结构版本、文件校验和与媒体引用。
5. 切换数据并运行迁移与健康检查。
6. 失败时恢复至操作前状态。

恢复功能必须通过真实隔离恢复演练验收，不能只验证备份文件存在。

## 14. 异常处理

- API 错误使用稳定错误码和可理解的中文说明，不向前端泄露堆栈和内部路径。
- 表单验证同时提供字段级提示和整页操作结果。
- 后台任务记录尝试次数、下次执行时间和经过脱敏的最后错误。
- 媒体转换失败时保留原上传记录但不得进入可发布状态。
- 生成快照、检查或切换失败时维持旧版本并清理临时文件。
- 数据库、磁盘、邮件或异地存储异常达到阈值时进入通知中心并发送重要提醒。
- 所有可重试操作必须具备幂等标识，避免重复发布、重复留言或重复通知。

## 15. 测试与验收

### 15.1 自动化测试

- 领域规则和服务单元测试
- 数据库模型与迁移测试
- 管理和公开 API 测试
- 登录、TOTP、CSRF、会话、限速和权限安全测试
- 内容编辑、媒体转换、引用检查和清理测试
- 快照生成、原子发布、失败保持和回滚测试
- 留言、私信、评论、审核、通知与反滥用测试
- 统计汇总与隐私字段测试
- 备份、损坏检测、隔离恢复和回退测试
- 桌面、平板、手机浏览器端到端测试
- 公开仓库敏感信息检查

### 15.2 验收标准

- 现有主页的布局、动效、链接和移动端表现不得退化。
- 现有留言、回复、审核状态和联系方式密文完整迁移。
- 后台停止时，公开网站和已发布内容仍可访问。
- “编辑、自动保存、预览、检查、发布、回滚”完整链路可重复通过。
- 原图不可通过公开地址访问，派生图片尺寸正确，媒体删除不产生断图。
- 访客联系方式在所有公开 API、页面、统计和日志中均不可见。
- 发布中断、磁盘不足、邮件失败和备份损坏等失败场景有明确结果且不破坏当前数据。
- 真实备份能够在隔离环境恢复并通过完整性检查。
- 公开页面在常见手机、平板和桌面宽度下无横向溢出，首屏继续保持轻量。

## 16. 开发分期

### 阶段一：管理平台基础

建立后台应用框架、模块 API、站长账户、TOTP、恢复码、会话管理、通知中心和审计日志。保留现有留言板生产能力。

### 阶段二：内容与媒体

实现全部内容类型、科研模块、可视化与 Markdown 编辑器、媒体处理、SEO、搜索和现有公开内容迁移。

### 阶段三：页面与发布

实现结构化页面搭建器、三端预览、发布检查、静态快照、发布历史、差异和回滚。

### 阶段四：互动与运维

将留言后台扩展为统一互动中心，加入评论、渐进验证码、访问统计、完整通知、服务状态、备份恢复和加密异地备份。

每个阶段均需通过相应自动化测试和人工验收后再进入下一阶段。阶段完成只代表该阶段具备生产质量，不减少最终功能范围。

## 17. 迁移与兼容

- 使用 Alembic 增量迁移现有 SQLite，不重建或清空生产数据库。
- 在迁移前创建即时备份，并用生产数据副本完成一次迁移演练。
- 现有公开 URL、留言板地址和后台入口保持兼容；必须调整的地址使用永久重定向。
- 现有主页内容和媒体导入内容库后，生成结果需要与当前线上版本进行视觉对照。
- 新管理后台上线采用可回滚发布；旧后台在新后台验收通过前保持可恢复。

## 18. 公开仓库边界

设计、源代码、迁移、测试和示例配置可以进入公开仓库。以下内容禁止提交：

- 生产域名之外的服务器连接信息
- 密码、密钥、恢复码、会话、Cookie 和 SMTP 凭据
- 数据库、备份和操作日志
- 访客留言、私信、联系方式和统计明细
- 本机绝对路径、私钥文件名和部署时产生的敏感临时文件

公开仓库隐私检查必须作为提交前和发布前的固定测试。
